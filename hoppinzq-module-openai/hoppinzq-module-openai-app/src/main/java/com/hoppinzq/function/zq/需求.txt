MCP VS functionCall

1、MCP是一个多模态的AI助手，支持语音、文本、图片等多种输入方式，
能够处理复杂的任务和问题。它可以与用户进行自然语言对话，并根据上下文理解用户的意图。

2、functionCall是一个函数调用的接口，允许开发者在代码中定义函数，并通过API调用这些函数。
它主要用于处理特定的任务和操作，例如数据处理、计算等。

3、MCP开发非常简单，目前支持python、nodejs和java。
其中SpringAI为MCP提供了JavaSDK，用注解的方式来定义函数和声明参数，编写起来非常方便。
注解是一切封装的终极形态 --hoppinzq
部署也非常简单，只需要将jar包放到指定MCP应用或者发布到MCP平台即可。
由于SpringAI基于JDK17开发，需要程序员使用JDK17以上的版本。自建MCP Server的运行环境也需要JDK17以上的版本。

4、functionCall的开发相对复杂，需要开发者手动编写函数和实现细节，但是没有任何编程语言和版本的限制，
它的本质就是封装function请求体和解析响应体并手动调用函数，Java开发人员可以编写非常优雅的封装，并可以基于反射动态调用方法。

5、MCP的JavaSDK由于封装、导致了拓展并不方便，且调试和鉴权困难。
functionCall可以完美解决这些问题，仅仅对开发者的编码要求比较高。（比如加个检查、确认功能）

6、MCP几乎支持所有大模型，而functionCall目前只支持OpenAI的GPT-3.5和GPT-4模型。
